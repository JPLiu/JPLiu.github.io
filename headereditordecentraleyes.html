Header Editor 与 Decentraleyes 共用需要注意排除

最近在 Firefox 上尝试使用 Decentraleyes 这个扩展，在安装完扩展之后打开扩展的 测试页面 时，发现页面提示 Decentraleyes 工作异常。如下图：

看到这个提示第一反应是按照上图的提示看看 uBlock Origin 设置的动态规则，但尝试后发现即使把 uBlock Origin 的动态过滤功能关闭掉 Decentraleyes 在测试页面也一样提示工作异常。

通过禁用扩展排查，发现问题的成因在于 Header Editor，由于在 Header Editor 中，我使用了网上 dupontjoy 的规则，其中下图几条对 Google 重定向规则，与 Decentraleyes 的功能发生了冲突。

禁用上述规则之后再次到 Decentraleyes 的 测试页面 进行测试，则提示 Decentraleyes 正常工作。
